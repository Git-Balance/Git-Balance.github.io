Biotech Project (Diabetes Insipidus)

Link to project presentation

In 11th grade Biotech, I had to choose a biology related problem and find a solution using technology and skills learned in class. I chose treating Diabetes Insipidus, a condition where the body does not retain water properly (this was a condition I was tested for before I was tested for celiac disease). I came up with a protocol to create bacteria that generates vasopressin, a hormone that people with Diabetes Insipidus often do not produce properly, by inserting a plasmid with the vasopressin gene, then harvesting the vasopressin from the bacteria. This involves CRISPR technology. While originally this was a group project, I chose to work on this project on my own.

While working on the project, I realized that there not a way of detecting osmolarity levels in a person quickly enough to respond to changes in their environment. I suspect that hypothetically supraoptic nuclei cells, which detect osmolarity, could be grown and used to monitor osmolarity levels; I did not have the time to research this for my project.

Programming on Android

Link to GitHub Repository

Termux is an app on Android that can run a Linux terminal. This allows me to use desktop Linux software like Vim, Git, and cmake on Android. This let me program on Android (in this case, on Kindle Fire).

Termux does not support graphical applications by default. I had to create a VNC server inside of Termux, start a window manager (like i3), then use a VNC client app on Android to connect to the server inside of Termux.

Nix and NixOS

  • NixOS configuration on GitHub
  • GitHub link for latino on Nix

NixOS is a Linux distribution that uses the Nix language to configure itself and install/configure packages. I used this as my OS on my laptop for almost a year. I used it because it could be configured almost entirely through code; it made package management simpler. I stopped using it because it could only be configured through the Nix language; Nix introduced a level of abstraction that made it hard to debug problems and make changes quickly. Additionally, the things I learnt about Nix and NixOS could not be applied outside of the Nix ecosystem.

On NixOS, I needed to add extra configuration to my programming projects; many of the projects here like Raylib Sand include a shell.nix file.
